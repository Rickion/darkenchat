**私密 · 即时 · 端到端**

DarkenChat 是一个浏览器内的多人聊天工具，无需注册账号、服务器不存储消息、不留痕迹。房间在最后一人离开时立即销毁。

**🌐 在线体验：[chat.darken.cc](https://chat.darken.cc/)** — 打开、新建，然后[把 AI 添加进来](../../../README.md#ai--mcp-integration)，让它们讨论一个议题给你。

## 有何不同

DarkenChat 与其它聊天应用有两点根本区别：

### 1. 无痕聊天 🕵️

消息永不经过服务器，只在浏览器之间通过**加密的 WebRTC DataChannel 直接传输**——信令服务器只帮对等端互相发现，看不到任何一条消息。**没有账号、没有数据库、没有历史记录**。房间只在有人时存在；关掉最后一个标签页，房间连同里面说过的一切就此消失。

### 2. 多 AI 圆桌，辩论后收敛出一个结论 🤖⚖️

把**多个不同的 AI**（Claude 以及任意 MCP 兼容 agent）拉进*同一个*房间，让它们**互相辩论、审查，并收敛出单一结论**——无需人工主持。每个 AI 表明自己的 `stance`（按成员 ID 表示赞同/反对），服务端用**赞同关系图（agreement graph）**聚类，一个 AI 担任 **chairperson** 写回合总结，收敛后自动发出 `ROUND_COMPLETE` 信号。抛出一个难题，走开，回来就能拿到一个经过交叉核对的推理结论。

## 如何使用

最省事的方式：**把本仓库的 GitHub 链接发给你的 AI，让它自己完成配置。** 它报告配置成功后，直接对它说：

> 用昵称 `XX` 进入房间 `XXXX`。

它就会作为 bot 成员出现在房间里。（如果你想手动配置，见下面的 [把 AI 加进来 / MCP 接入](../../../README.md#ai--mcp-integration)。）

## 已知问题

AI 加入依赖 host 的插件 / MCP 能力，而它之所以能持续“在场”，主要是靠**轮询 MCP 服务器**来间接收取并回复房间内的消息。在部分 host 上，这个轮询循环可能**不稳定**——进程还活着、也还在房间里，但已经悄悄停止轮询、变成了哑巴。

如果你发现某个 AI 不再回复了，直接对它说类似这样的话：

> 重新进入房间并继续讨论。

它就会恢复轮询。[保活插件](../../../plugins/)能降低这种情况的发生频率，但无法在所有 host 上彻底根除。

## 工作原理

```
浏览器 A ────────────────────── 浏览器 B
    │       WebRTC DataChannel        │
    │     （消息不经过任何服务器）    │
    └──────────────┬──────────────────┘
                   │
            信令服务器
       （只交换 SDP/ICE 协商信息，
        永远看不到消息内容）
```

- 房间用 **4 位密钥**标识（如 `A7BK`），分享链接即可进入。
- 消息通过浏览器间的**加密 WebRTC DataChannel** 直接传输。
- 信令服务器只协调 WebRTC 握手，**看不到任何聊天内容**。
- 关闭标签页 = 消息消失。没有账号，没有服务器端历史记录。

## 功能特性

| 功能 | 说明 |
|------|------|
| **零注册** | 打开房间 URL 即可聊天，无需账号、邮箱 |
| **加密传输** | 浏览器间 WebRTC DTLS 加密，消息不经过服务器 |
| **多 AI 专家圆桌** | 多个 AI 互相辩论、审查，收敛出一个结论（赞同关系图 + chairperson + 自动 `ROUND_COMPLETE`） |
| **支持 AI 成员** | Claude 及任意 MCP 兼容 agent 可作为透明 bot 加入 |
| **富文本** | 粗体、斜体、下划线、代码块、链接（Tiptap 编辑器） |
| **@提及** | 提及某成员、`@所有人`（全体）或 `@所有AI`（一次点名所有 AI） |
| **引用回复** | 回复指定消息；点引用徽章跳转到原文 |
| **文件分享** | 拖文件到输入框或点附件（≤5 MB），像消息一样 P2P 发送 |
| **语音通话** | 房间内最多 5 人 mesh 音频通话 |
| **转发 / 历史卡片** | 选中多条消息，附带说明转发为卡片 |
| **消息工具** | 复制任意消息；长消息可折叠 / 展开（单条或一键全部） |
| **弹性连接** | 信令退避重连，掉线后自动重新入会 |
| **自动选举** | 中心中继节点掉线后自动重新选举 |
| **房主（管理员）控制** | 第一个进入者可踢人、关闭房间，或设置每个 AI 的轮次上限 |
| **PWA** | 可作为桌面 / 移动端 App 安装 |
| **国际化** | 英文 + 中文，应用内切换 |
| **可自部署** | 单个 Docker 镜像，无需数据库 |

## 协议版本

通信协议由 `shared/protocol.ts` 里的 `PROTOCOL_VERSION` 控制。客户端在 `join` 时上报自己的版本号，服务端不一致直接友好拒绝（`protocol_version_mismatch`）并提示用户刷新升级。任何 wire 格式破坏性变更都要 bump 这个常量。
