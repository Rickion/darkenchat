# DarkenChat

**阅读其它语言版本：** [English](../../../README.md) · **中文**

**私密 · 即时 · 端到端**

DarkenChat 是一个浏览器内的多人聊天工具，无需注册账号、服务器不存储消息、不留痕迹。房间在最后一人离开时立即销毁。

---

## 工作原理

```
浏览器 A ────────────────────── 浏览器 B
    │       WebRTC DataChannel        │
    │     （消息不经过任何服务器）    │
    └──────────────┬──────────────────┘
                   │
            信令服务器
       （只交换 SDP/ICE 协商信息，
        永远看不到消息内容）
```

- 房间用 **4 位密钥**标识（如 `A7BK`），分享链接即可进入。
- 消息通过浏览器间的**加密 WebRTC DataChannel** 直接传输。
- 信令服务器只协调 WebRTC 握手，**看不到任何聊天内容**。
- 关闭标签页 = 消息消失。没有账号，没有服务器端历史记录。

## 功能特性

| 功能 | 说明 |
|------|------|
| **零注册** | 打开房间 URL 即可聊天 |
| **加密传输** | 浏览器之间的 WebRTC DTLS 加密 |
| **富文本** | 粗体、斜体、下划线、代码块、链接（Tiptap 编辑器） |
| **语音通话** | 房间内最多 5 人 mesh 音频通话 |
| **转发 / 历史卡片** | 选中多条消息，附带说明转发为卡片 |
| **自动选举** | 中心节点掉线后自动重新选举 |
| **房主（管理员）控制** | 第一个进入者可以踢人或关闭房间 |
| **支持 AI 成员** | Claude 等 MCP 兼容 agent 可作为透明 bot 加入 |
| **PWA** | 可作为桌面 / 移动端 App 安装 |
| **国际化** | 英文 + 中文 |
| **可自部署** | 单个 Docker 镜像，无需数据库 |

## 协议版本

通信协议由 `shared/protocol.ts` 里的 `PROTOCOL_VERSION` 控制。客户端在 `join` 时上报自己的版本号，服务端不一致直接友好拒绝（`protocol_version_mismatch`）并提示用户刷新升级。任何 wire 格式破坏性变更都要 bump 这个常量。
